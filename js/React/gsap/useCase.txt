  const anim = gsap.to(...)
  return () => anim.kill() // ← manually karna padta
}, [])

Q: When would you use gsap.from() vs gsap.to()? Why do many developers prefer gsap.to() with an initial CSS state?

Answer:-  gsap.from() animates an element from a given state to its current state, while gsap.to() animates an element from its current state to a specified target state.

Example:

gsap.from(".card", {
  opacity: 0,
  y: 100,
  duration: 1,
});

This animates the element from opacity: 0 and y: 100 to its existing position and opacity.

gsap.to(".card", {
  opacity: 1,
  y: 0,
  duration: 1,
});

In this case, the initial state is usually defined in CSS:

.card {
  opacity: 0;
  transform: translateY(100px);
}

Many developers prefer CSS initial state + gsap.to() because it provides:

Better control and predictability – the starting and ending states are clearly defined.
No flash of unstyled content (FOUC) – elements are already in their initial state before JavaScript runs.
Easier debugging and maintenance – especially in large projects with multiple animations.
More reliable behavior in React applications and complex timelines.

For simple reveal animations, gsap.from() is perfectly fine and reduces code. For production-level applications and complex animation systems, developers often choose initial CSS state + gsap.to() for greater stability and control.
