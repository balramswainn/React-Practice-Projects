What is a cleanup function in React?
A cleanup function in React is a function returned from useEffect that runs before the component unmounts or before the effect runs again. It is used to remove side effects such as event listeners, timers, subscriptions, or WebSocket connections to prevent memory leaks and unnecessary executions.


useEffect(() => {
  const handleResize = () => {
    console.log(window.innerWidth);
  };

  window.addEventListener("resize", handleResize);

  return () => {         //ye cleanup hai
    window.removeEventListener("resize", handleResize);
  };
}, []);


In this example, an event listener is attached when the component mounts. When the component unmounts, the cleanup function removes the listener. Without cleanup, the listener would remain active in memory and continue responding to resize events, which can cause memory leaks and performance issues.
A cleanup function is the function returned from useEffect. It is used to clean up side effects like event listeners, intervals, timers, and subscriptions when a component unmounts or before the effect re-runs, helping prevent memory leaks.



memory leak: In React, a memory leak occurs when a component retains references to objects or background tasks after it unmounts, preventing the browser's JavaScript garbage collector from reclaiming that memory. This gradually drains available RAM, slowing down your application and causing browser tabs to crash.

Interval : const id = setInterval(...)
Cleanup: clearInterval(id)

Timeout : const id = setTimeout(...)
Cleanup: clearTimeout(id)

WebSocket : const socket = new WebSocket(...)
Cleanup: socket.close()
